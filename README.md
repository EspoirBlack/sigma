# sigma
SIGMA stands for Security Information Gathering and Monitoring Assistant, it is a security tool built for junior soc analyst.
The proposed system operates by continuously collecting and analyzing windows event logs from the security, system and application logs, while simultaneously monitoring running processes and active network connections on the host system. Unlike complex enterprise SIEM solutions that require extensive infrastructure, SIGMA is designed as a lightweight, deployable application that provides immediate security visibility on individual windows workstations. The system employs a rule-based detection engine comprising five core security rules targeting common attack vectors: multiple failed login attempts (indicating of brute force attacks), suspicious process execution, unusual port connections (potential backdoors), privilege escalation events and unauthorized service installations. Each detection rule maps directly to specific windows event IDs and behavioral patterns that junior analyst would typically investigate during routine monitoring duties.
A key capacity of SIGMA is its integration with external threat intelligence through the VirusTotal API, enabling the system to enrich detected indicators of compromise with global reputation data. When suspicious network connections are identified, the system automatically extracts remote IP addresses and queries VirusTotal’s database to determine whether those addresses have been associated with known malicious activity. This capability mirrors the investigative process a junior analyst would follow when triaging alerts, manually checking indicators against threat intelligence platforms to validate suspicious and prioritize responses. The integration demonstrates how even a rule-based system can leverage external intelligence sources to provide context that would otherwise require manual research.  
The system provides an intuitive web-based interface developed using the Flask framework, allowing users to interact with the system through natural language queries such as “show failed logins” or “check suspicious processes”. This query functionality is implemented through keyword matching and log filtering rather than artificial intelligence, providing an accessible way for users to investigate security events without needing to understand complex query languages. The dashboard presents real-time system status, active alerts color-coded by severity and live log feeds, enabling continuous visibility into the security posture of the host. Users can generate comprehensive PDF reports containing executive summaries, system statistics, detailed alert listings and actionable recommendations supporting compliance requirements and incident documentation needs.
 All alerts and system events are persistently stored in JSON format, ensuring data survives application restarts and can be reviewed historically. The application includes comprehensive error handling and logging capabilities, with clear user guidance when administrative privileges are required for log access. The system is built entirely using python and open-source libraries, demonstrating that effective security monitoring capabilities can be developed and deployed without reliance on expensive commercial solutions or complex artificial intelligence systems.
SIGMA successfully demonstrate how automated log collection, rule-based detection and threat intelligence integration can reduce the workload on junior SOC analysts by handling routine monitoring tasks, correlating events across multiple data sources and providing enriched context for security decisions. The system achieves its objectives through practical implementation of windows log collection, real-time system monitoring, rule-based detection, external threat intelligence integration and professional reporting capabilities.
